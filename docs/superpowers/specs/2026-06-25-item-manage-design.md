# 物品整理 App 设计文档

**日期**: 2026-06-25
**平台**: iOS
**最低版本**: iOS 17+
**Bundle ID**: com.wind.ItemManage
**远程仓库**: https://gitee.com/WLL1011/item-manage
**当前实现范围**: Phase 1 + Phase 2

## 项目概述

一个用于记录和管理家庭物品的 iOS 应用，核心功能包括：
- 记录物品的存放位置和买入价格
- 追踪物品的使用时间、用完时间、过期时间
- 自动计算每日使用成本
- 支持搜索和按区域分类

## 技术方案

### 技术栈
- **UI 框架**: SwiftUI
- **数据持久化**: SwiftData (iOS 17+)
- **架构模式**: 简化 MVVM
- **数据存储**: 仅本地存储

### 架构选择理由
1. **SwiftUI**: 声明式语法，开发效率高，适合标准 UI 组件
2. **SwiftData**: 苹果原生方案，与 SwiftUI 无缝集成，代码量少
3. **本地存储**: 简单可靠，无需维护后端服务

## 数据模型设计

### Item（物品）实体

| 属性 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | UUID | 是 | 主键 |
| name | String | 是 | 物品名称 |
| purchasePrice | Decimal | 是 | 买入价格（≥ 0） |
| startDate | Date | 是 | 开始使用时间 |
| endDate | Date? | 否 | 用完时间 |
| expiryDate | Date? | 否 | 过期时间 |
| area | Area | 是 | 所属区域（关系） |
| specificLocation | String | 是 | 具体位置描述 |
| createdAt | Date | 是 | 创建时间 |
| updatedAt | Date | 是 | 更新时间 |

**计算属性**:
- `usedDays`: Int - 已使用天数
- `dailyCost`: Decimal - 每日使用成本
- `isExpired`: Bool - 是否已过期
- `isUsedUp`: Bool - 是否已用完

### Area（区域）实体

| 属性 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | UUID | 是 | 主键 |
| name | String | 是 | 区域名称 |
| items | [Item] | - | 该区域下的所有物品（关系） |
| createdAt | Date | 是 | 创建时间 |

### 实体关系
- Area ← (一对多) → Item
- 删除区域时，若该区域下有物品，弹窗让用户三选一：**迁到其他区域** / **删除全部物品** / **移至「未分类」**

### 已确认决策（2026-06-25）

| 主题 | 决策 |
|------|------|
| 实现范围 | Phase 1 + Phase 2（不含 Phase 3 无障碍/性能专项） |
| 状态优先级 | 已用完 > 已过期；标记用完后不再显示过期标识 |
| 即将过期 | 列表卡片显示黄色警告图标（7 天内） |
| 「未分类」 | 系统保留、不可删除；不出现在添加/编辑的区域 Picker 中 |
| 列表排序 | 默认按名称；导航栏 Menu 支持字段 + 升/降序 |
| 排序字段 | 名称、创建时间、每日成本、买入价格、开始使用时间 |
| 区域筛选 | 导航栏 Menu / Picker，可选「全部」或某区域 |
| 空区域 | 首页始终显示所有区域（含 0 件） |
| 下拉刷新 | 重新加载 SwiftData 查询 |
| 过期时间校验 | 不做额外校验 |
| 首次引导 | 多页 onboarding；区域管理 Tab 导航栏「帮助」可再次打开 |

## 核心功能模块

### 1. 物品列表（首页）

**显示内容**:
- 按区域分组展示物品
- 每个物品卡片显示：
  - 物品名称
  - 位置信息（区域 + 具体位置）
  - 每日使用成本（橙色大字）
  - 状态标识：已用完（灰色）> 已过期（红色）> 即将过期（黄色，7 天内）
- 空状态提示（基础文案，Phase 3 再完善插画）

**交互功能**:
- 点击物品 → 进入详情页
- 右上角 "+" 按钮 → 添加新物品
- 左滑删除物品
- 导航栏 Menu：按区域筛选（「全部」或某区域）
- 导航栏 Menu：排序（字段 + 升/降序，默认名称升序）
- 下拉刷新（重新加载 SwiftData 查询）
- 显示所有区域分组（含 0 件空区域）

### 2. 物品详情页

**显示内容**:
- 基本信息：名称、价格
- 位置信息：区域、具体位置
- 时间信息：开始使用、用完时间、过期时间
- 成本统计：已使用天数、每日成本
- 状态提示：过期状态、用完状态

**交互功能**:
- 编辑按钮 → 进入编辑表单
- 快速标记为已用完（填写用完时间为今天）
- 删除物品（带确认）

### 3. 添加/编辑表单

**表单字段**:
1. 物品名称（文本输入）
2. 买入价格（数字输入，支持小数）
3. 选择区域（Picker 选择器，可快速新建区域；「未分类」不在选项中）
4. 具体位置（文本输入）
5. 开始使用时间（日期选择器，默认今天）
6. 用完时间（可选，日期选择器）
7. 过期时间（可选，日期选择器）

**表单验证**:
- 名称不能为空
- 价格必须 ≥ 0
- 必须选择区域
- 开始时间不能晚于用完时间

**表单分段**:
- 基本信息：名称、价格
- 位置信息：区域、具体位置
- 时间信息：开始使用、用完、过期

### 4. 区域管理

**功能列表**:
- 查看所有区域及每个区域的物品数量
- 添加新区域
- 重命名区域
- 删除区域（有物品时弹窗三选一：迁到其他区域 / 删除全部物品 / 移至「未分类」）
- 导航栏「帮助」按钮 → 再次打开多页引导

**预置区域**:
首次启动自动创建：客厅、卧室、厨房、卫生间、储藏室

**系统保留区域**:
- "未分类"：不可删除，用于容纳删除区域后的孤立物品

### 5. 搜索功能

**搜索范围**:
- 物品名称
- 区域名称
- 具体位置描述

**搜索特性**:
- 实时搜索（输入即搜索）
- 支持模糊匹配（部分关键词匹配）
- 显示匹配结果的高亮
- 可按价格、使用时间排序

**实现方案**:
使用 SwiftData 的 `@Query` 和 Predicate 组合查询条件：
```
名称包含关键词 OR 区域名称包含关键词 OR 具体位置包含关键词
```

## 关键算法设计

### 每日使用成本计算

**计算公式**:

```swift
如果已用完（endDate 有值）：
  usedDays = endDate - startDate 的天数
  dailyCost = purchasePrice / usedDays

如果使用中（endDate 为空）：
  usedDays = 今天 - startDate 的天数
  dailyCost = purchasePrice / usedDays
```

**特殊情况处理**:
- 当天购买当天使用：天数按 1 天计算（避免除以 0）
- 价格为 0：每日成本显示为 ¥0.00
- 日期计算精确到天（忽略时分秒）

**显示格式**:
- 金额保留 2 位小数
- 货币符号：¥（人民币）
- 示例：¥12.50/天

### 日期处理规范

**时区**:
- 统一使用设备本地时区
- 存储时使用 Date 类型（UTC）
- 显示时转换为本地时区

**天数计算**:
```swift
Calendar.current.dateComponents([.day], from: startDate, to: endDate).day ?? 0
```

**过期判断**:
- 即将过期：expiryDate 在未来 7 天内
- 已过期：expiryDate < 今天

## 风险与限制

### 技术限制
- 需要 iOS 17+，无法支持更老设备
- 仅本地存储，换设备数据不同步

### 数据安全
- 本地数据未加密（未来可考虑加密）
- 用户需自行备份设备

### 性能限制
- 超大量数据（10000+ 条）可能影响性能
- 复杂搜索查询需要优化

本设计方案采用 SwiftUI + SwiftData 现代技术栈，提供了清晰的两层位置管理、自动成本计算、全文搜索等核心功能。架构简洁易扩展，适合快速开发和迭代。

核心特性：
✅ 两层位置管理（区域 + 具体位置）
✅ 自动计算每日使用成本
✅ 灵活的时间字段（必填+可选）
✅ 全文搜索功能
✅ 原生技术栈，性能优秀
✅ 简洁架构，易于维护
